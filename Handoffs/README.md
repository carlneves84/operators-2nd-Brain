# Handoffs

Session transfer documents, generated by the `handoff` skill.

A handoff is not a summary of what happened. It is a starting kit for what comes next. The reader is not looking back -- they are starting forward.

Files are named `handoff-YYYY-MM-DD-[slug].md` where the slug is a 2-4 word kebab-case label for the work.

Examples:
- `handoff-2026-03-15-activation-audit.md`
- `handoff-2026-04-02-pipeline-debug.md`
- `handoff-2026-05-20-leadership-deck.md`
