# Projects

One file per active initiative. Named `[Initiative Name].md`.

The `weekly-planner` skill creates stubs here automatically when it finds a `[[cross-reference]]` in your weekly note that does not have a matching file.

## Template

```markdown
# [Initiative Name]

**Status:** Active / On hold / Killed
**First referenced:** YYYY-MM-DD
**Weekly note:** YYYY-MM-DD

## Goal
[What success looks like in one sentence]

## Open Questions
- [question]

## Key Decisions
- **YYYY-MM-DD:** [Decision made and why]

## Next Action
[Single most concrete next step]
```

## What goes here vs. Context/

Projects are initiative-specific: one file, one initiative, updated as the work evolves. When a project is killed or completed, move it to an `Archive/` subfolder rather than deleting -- the decision history is worth keeping.

Context/ files are program-wide: learnings, metric baselines, strategic direction. They apply across all projects.
